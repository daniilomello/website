title: "Trabalhando com alias no Git"
publishedAt: "2024-06-18"
---

Eu gosto de utilizar comandos simplificados quando eu estou trabalhando com Git, então eu criei alguns alias que acredito serem úteis no dia a dia.

Comece acessando as configurações do Git:

```bash
git config --global --edit
```

Primeiro alias que eu gosto de utilizar é o `git c "commit description"` que serve para adicionar todos os itens ao staged e commitar eles ao mesmo tempo.

```bash
c = !git add --all && git commit -m
```

O segundo é o `git s`, que serve para verificar o status de uma forma mais reduzida.

```bash
s = !git status -s
```

O próximo é o `git log`, porém de uma forma mais organizada e limpa, trazendo a hash reduzida do commit, a branch e descrição do commit, junto com o nome da pessoa e data do commit.

```bash
l = !git log --pretty=format:'%C(blue)%h%C(red)%d %C(white)%s - %C(cyan)%cn, %C(white)$cr'
```

Outro alias que eu gosto é o `git amend`, que junta o seu commit atual ao anterior, excelente para momentos que você esquece de alterar e commitar um arquivo por exemplo.

```bash
amend = !git add --all && git commit --amend --no-edit
```

Para quem trabalhada com commit semântico, `git count` é um alias útil para validar histórico de commits.

```bash
count = !git shortlog -s --grep
```

Por último, são lias para criar, deletar e acessar branches.

```bash
b = !git checkout
bc = !git checkout -b
bl = !git branch
bd = !git branch -D
```
